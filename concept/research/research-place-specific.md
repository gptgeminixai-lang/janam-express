SUMMARY:
Every technical building block for a place-of-birth "time machine" page exists free of charge. Place selection: the community India Post API (api.postalpincode.in, GET /pincode/{PINCODE} and /postoffice/{NAME}, no key, ~1000 req/hr/IP) resolves any of ~19,000 pincodes to district/state, and the official data.gov.in "All India Pincode Directory" CSV adds latitude/longitude per post office (~154k rows) — so a user can type either a pincode or a city name (Open-Meteo's free geocoding API or GeoNames IN.zip both handle Indian city autocomplete with lat/long). With lat/long in hand, two fully deterministic/free layers unlock: Open-Meteo's archive API (archive-api.open-meteo.com/v1/archive) returns the actual daily max/min temperature and rainfall for any Indian coordinate back to 1940 with no API key, and the SunCalc JS library computes sunrise/sunset/moon-phase client-side for the exact birth moment; the mhah-panchang npm package computes tithi/nakshatra ("birth star") offline, with ProKerala's panchang API (free 5,000 credits/month) as a richer fallback.

The emotional payload comes from three curated datasets that are small enough to ship as static JSON: (1) city renamings with dates — anyone born in Bombay before Nov 1995, Madras before Aug 1996, Calcutta before 2001, Bangalore before Nov 2014, Gurgaon before 2016 or Allahabad before 2018 was literally born in a differently-named city; (2) Census of India decadal city populations (1951–2011, from censusindia.gov.in tables / citypopulation.de compilations) enabling "Bangalore had 1.66M people when you were born; it has ~14M today — the city is 8x bigger"; (3) per-city milestone anchors ("your city had no metro, no mall, no McDonald's, no internet when you were born") built from Wikipedia's per-city timeline pages plus hard national dates: first mobile call 31 Jul 1995 (Kolkata), public internet 15 Aug 1995 (VSNL), first KFC Jun 1995 (Bangalore), first McDonald's 13 Oct 1996 (Delhi), Kolkata Metro 1984, Delhi Metro 24 Dec 2002, Namma Metro 20 Oct 2011, Chennai Metro 29 Jun 2015, Hyderabad Metro Nov 2017, new Bangalore & Hyderabad airports both 2008, Delhi T3 Jul 2010. The recommended architecture: pincode/city autocomplete → lat/long → live Open-Meteo weather call + client-side SunCalc/panchang + static JSON lookups for renaming, population interpolation, and city-decade anchors.

ITEMS:
## Place selection: api.postalpincode.in (free India Post community API)
Two endpoints, no auth key: GET https://api.postalpincode.in/pincode/{PINCODE} and GET https://api.postalpincode.in/postoffice/{BRANCHNAME}. Returns post office name, district, state, circle. Rate limit ~1000 requests/hour/IP (429 beyond that). Community-run (not an official SLA), so cache results or bundle a static pincode file as fallback. Alternative open-source clone: aniket-thapa.github.io/india-pincode-api.
SOURCE: https://api.postalpincode.in / http://www.postalpincode.in/Api-Details
WOW: User types the 6-digit pincode printed on their birth certificate address and the site instantly knows their taluk, district and state.

## Pincode-to-lat/long: official data.gov.in All India Pincode Directory CSV
Dataset 'All India Pincode Directory (contact details along with latitude and longitude)' — ~154,000 post office rows with columns CircleName, RegionName, DivisionName, OfficeName, Pincode, OfficeType, Delivery, District, StateName, Latitude, Longitude. Direct CSV historically at data.gov.in/files/ogdpv2dms/s3fs-public/dataurl03122020/pincode.csv; also mirrored on Kaggle ('India Pincode with Lat-Long Data') and GitHub (dropdevrahul/pincodes-india, avinashcelestine/Pincodes-data which maps pincodes to Census 2011 codes). Ship this as a static lookup — no runtime API dependency.
SOURCE: https://www.data.gov.in/resource/all-india-pincode-directory-contact-details-along-latitude-and-longitude
WOW: Every birth place in India, down to village post offices, resolves to coordinates offline — the key that unlocks weather, sunrise and panchang for the exact spot.

## City-name autocomplete with coordinates: Open-Meteo Geocoding API + GeoNames
GET https://geocoding-api.open-meteo.com/v1/search?name=Bengal&count=10&country_code=IN — free, no key, fuzzy/partial matching, returns lat/long, admin1 (state), timezone, population. Offline alternatives: GeoNames dump download.geonames.org/export/dump/IN.zip (CC-BY, all populated places in India) or cities1000.zip; Simplemaps free India tier (~354 cities CSV); Kaggle '4000+ Indian Cities: Names, States, Lat, Lon'.
SOURCE: https://open-meteo.com/en/docs/geocoding-api ; https://www.geonames.org/export/
WOW: Type-ahead 'Kozhi…' finds Kozhikode with coordinates and timezone in one keystroke round-trip, zero cost.

## Actual weather on the birth date: Open-Meteo Historical (Archive) API, data back to 1940
GET https://archive-api.open-meteo.com/v1/archive?latitude=12.97&longitude=77.59&start_date=1987-06-15&end_date=1987-06-15&daily=temperature_2m_max,temperature_2m_min,precipitation_sum,weathercode&timezone=auto. ERA5 reanalysis (0.25 deg, from 1940) + ERA5-Land (0.1 deg, from 1950). No API key for non-commercial use (free tier commonly cited as <10k calls/day); data CC-BY 4.0; commercial plans exist. Covers all of India gap-free — works for 1950s grandparents too. IMD's own data (cdsp.imdpune.gov.in; gridded daily rainfall 0.25 deg 1901-2024, temperature 1x1 deg 1951-2024) is authoritative but binary-file downloads with registration, not a live API — use IMD only if you want pre-1940 rainfall.
SOURCE: https://open-meteo.com/en/docs/historical-weather-api ; https://cdsp.imdpune.gov.in/
WOW: 'It rained 34 mm in Chennai the day you were born, and the high was 31.2 C' — a real, verifiable fact about their day zero.

## City population then vs now: Census of India decadal data 1951-2011
Official source: censusindia.gov.in census tables (Table A-2 'Decadal variation in population 1901-2011', A-4 towns and urban agglomerations); NADA catalog entry 43333. Easiest machine-readable compilation: citypopulation.de/en/india/cities/ (every city/UA >150k with 1991/2001/2011 and older censuses), Wikipedia per-city demographics, CEIC 'Census: Population: by Selected Cities'. Example numbers to bake in: Bangalore 1.66M (1971) → 4.1M (1991) → 8.4M city / 8.5M UA (2011) → ~14M today, i.e. a person born there in 1971 has seen the city grow ~8x. Since 2021 census never happened, interpolate 1951-2011 decadal points and extrapolate to today; store as a static JSON of the top ~500 towns.
SOURCE: https://censusindia.gov.in/census.website/data/census-tables ; https://www.citypopulation.de/en/india/
WOW: 'The Bengaluru you were born into had fewer people than today's Jaipur' — concrete scale of how much their hometown exploded within their lifetime.

## Sunrise/sunset/moon phase: SunCalc JS — zero-API, fully client-side
npm install suncalc (mourner/suncalc, ~68k weekly downloads, BSD-2). SunCalc.getTimes(date, lat, lng) → sunrise, sunset, dawn, dusk, goldenHour; SunCalc.getMoonIllumination(date) → phase (0-1, 0=new, 0.5=full) and illuminated fraction; SunCalc.getMoonTimes() → moonrise/set. Based on Jean Meeus 'Astronomical Algorithms', matches timeanddate.com accuracy. Works for any historical date deterministically — no rate limits ever. Maintained forks: suncalc3. For higher precision use astronomy-engine npm package.
SOURCE: https://github.com/mourner/suncalc ; https://www.npmjs.com/package/suncalc
WOW: 'The sun rose at 6:04 AM over Patna the morning you were born, under a waxing gibbous moon, 83% lit' — computed instantly in the browser, offline.

## Birth star / nakshatra: mhah-panchang (free JS lib) + ProKerala API fallback
Client-side: npm 'mhah-panchang' computes Tithi, Nakshatra, Karana, Yoga, Raasi, Paksha, Masa, Ritu from date+lat/long — calendar(date, lat, lng) method; zero cost, works for any historical date; also 'panchangJS' (schenna/panchangJS) and '@bidyashish/panchang'. Hosted option: ProKerala Astrology API (api.prokerala.com) — free tier 5,000 credits/month, 5 req/min, OAuth2 client-credentials; panchang endpoint returns nakshatra, tithi, yoga, karana for date+coordinates; paid Ruby plan Rs 999/mo for 100k credits. Recommendation: compute with mhah-panchang locally (no rate limit for viral traffic), validate output once against ProKerala/DrikPanchang.
SOURCE: https://www.npmjs.com/package/mhah-panchang ; https://api.prokerala.com/pricing
WOW: 'Your janma nakshatra is Rohini, born in Shukla Paksha Tritiya of Ashadha masa' — the detail Indian parents and grandparents will immediately recognize and share.

## Recommended data architecture for the place feature
Flow: (1) user types city or pincode → autocomplete from bundled data.gov.in pincode CSV + GeoNames IN cities (static, indexed client-side or in SQLite); (2) resolved lat/long + birth date → one Open-Meteo archive call (only live API needed; cache by pincode+date); (3) SunCalc + mhah-panchang run in browser; (4) static JSON lookups: renames.json (~30 rows), city_population.json (top 500 towns x 7 census years from citypopulation.de/censusindia), city_milestones.json (top 15 metros x ~10 dated anchors from Wikipedia city timelines). Everything except the weather call is deterministic and offline, so per-page cost is effectively zero and the page still mostly works if Open-Meteo is down.
SOURCE: Synthesis of: api.postalpincode.in, data.gov.in, geocoding-api.open-meteo.com, archive-api.open-meteo.com, github.com/mourner/suncalc, npmjs.com/package/mhah-panchang, en.wikipedia.org/wiki/Renaming_of_cities_in_India, citypopulation.de
WOW: One free live API plus three small static JSON files produce a page with real weather, real astronomy, real panchang, real demographics and a name-of-city plot twist — all personalised to a village-level birthplace.
